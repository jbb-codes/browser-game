06 — Listening to the Player

Tracks which keys are held down at any moment using a simple dictionary.

What this file does

Maintains a keys dictionary — an object where each key name maps to true (currently held) or false (released).

Listens for keydown events — when a key is pressed, it sets that entry to true.

Listens for keyup events — when a key is released, it sets that entry to false.

isDown(key) is called by Player.update() every frame to check whether a specific key is currently held.

dispose() removes the event listeners when the game shuts down — prevents memory leaks.

How It Works — Event Flow

⌨️
Player presses a key
The browser fires a keydown event on the window containing the key name.
📥
onKeyDown handler fires
keys[e.key.toLowerCase()] = true — stores the lowercased key as true in the dictionary.
🔍
Player.update() calls isDown("w") each frame
Returns keys["w"] === true. While the key is held, this returns true every frame and the player keeps moving.
📤
Player releases the key
Browser fires keyup. Handler sets keys["w"] = false. isDown returns false and movement stops next frame.
