05 — The Player Character

A box mesh that moves relative to wherever the camera is pointing.

What this file does

Creates a 1×2×1 box mesh (the player body). Positioned at Y=1 so its bottom edge sits exactly on the ground at Y=0.

Applies a purple material (Color3(0.6, 0.4, 0.8)) so the player is easy to spot against the green ground.

In update() each frame, it reads the camera's forward direction, flattens it to the ground plane (removes the Y component), then derives a "right" direction using a cross product.

WASD and arrow keys both work — each pair maps to the same direction. The result is guarded with if (move.length() > 0) to avoid normalizing a zero vector (which would produce NaN), then normalized for equal speed in all directions and scaled by SPEED = 0.1 units per frame.

How Movement Direction is Calculated

Step 1
const forward = camera.getForwardRay().direction
Ask the camera which way it is looking. Returns a 3D vector pointing into the scene.
Step 2
const forwardFlat = new Vector3(forward.x, 0, forward.z).normalize()
Drop the Y component so the player only moves horizontally, not up or down hills. Then normalize to make it length 1.
Step 3
const right = Vector3.Cross(Vector3.Up(), forwardFlat).normalize()
The cross product of Up(0,1,0) and Forward always gives a vector pointing 90° to the right. This is how "strafe right" is computed from the camera angle.
Step 4
move.normalize().scaleInPlace(SPEED)
Pressing W+D diagonally would produce a vector longer than 1 (faster than moving straight). Normalizing first makes all directions equal speed, then SPEED (0.1) is applied.
