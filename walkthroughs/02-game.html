02 — The Orchestrator

The class that owns everything and drives the update → render cycle.

What this file does

Game is the top-level class. It owns the scene, player, camera, and input — and holds them together for the entire game session.

The constructor builds all four subsystems in a specific order — scene must exist before player or camera can be created inside it.

It registers an onBeforeRenderObservable — a Babylon hook that fires just before each frame is drawn. This is where game logic runs.

update() tells the player to move based on input and camera direction. render() tells the scene to draw itself.

Constructor Build Order — Why It Matters

1
createScene(engine)
Scene must exist first — everything else lives inside it
2
new InputManager()
Doesn't need the scene — just starts listening to the keyboard
3
new Player(scene)
Creates a mesh inside the scene — scene must already exist
4
new GameCamera(scene, canvas, player.mesh)
Needs both the scene and the player's mesh to orbit around

The Frame Cycle — Every ~16ms

onBeforeRender fires
→
update()
→
scene.render()

Babylon fires the observable, your logic runs, the scene draws. Repeats ~60× per second.
